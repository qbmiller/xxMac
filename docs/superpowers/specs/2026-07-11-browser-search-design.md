# 浏览器书签与历史搜索设计

## 目标

在 xxMac 启动器中增加 Chrome 和 Microsoft Edge 的书签、历史记录搜索。首次使用时根据 macOS 默认浏览器选择 Chrome 或 Edge，之后允许用户在设置中覆盖。书签和历史关键词默认分别为 `bm`、`bh`，并支持自定义。

首版仅支持当前浏览器最近使用的 Profile，不向用户暴露 Profile 选择。Safari 不在本期实现，但数据源接口必须允许以后增加 Safari，而不修改启动器主搜索流程。

## 用户体验

“搜索”工具的第二栏新增“浏览器搜索”。详情页包含：

- 启用浏览器搜索的开关。
- Chrome、Microsoft Edge 浏览器选择器。
- 书签搜索关键词输入框，默认 `bm`。
- 历史搜索关键词输入框，默认 `bh`。

首次没有保存配置时，xxMac 读取 macOS 默认 HTTP/HTTPS 浏览器：

- 默认浏览器是 Chrome 或 Edge 时选择对应浏览器。
- 无法识别默认浏览器时，优先选择已安装的 Chrome，其次选择已安装的 Edge。
- 两者均未安装时仍保留配置页，但启动器不返回浏览器结果，并在设置页显示未安装状态。

用户手动选择浏览器后，选择写入 `preferences.json`，不再随系统默认浏览器变化。

启动器输入规则：

- `bm 查询词`：按标题和 URL 搜索当前 Profile 的书签。
- `bh 查询词`：按标题和 URL 搜索当前 Profile 的历史记录。
- 只输入书签关键词时，展示一组书签候选。
- 只输入历史关键词时，按最近访问时间展示历史候选。
- 回车后明确使用设置中选择的浏览器打开 URL，而不是交给当前系统默认浏览器。

搜索结果显示页面标题、域名和所选浏览器图标。无结果、浏览器未安装或数据文件不可读时，不用模态弹窗打断用户；设置页显示持久状态，启动器使用简短的内联状态结果。

## 架构

### 浏览器定义

`BrowserKind` 枚举首版包含 Chrome 和 Edge，并集中维护显示名称、Bundle ID、应用路径候选及用户数据目录。以后增加 Safari 时扩展此枚举和数据源工厂。

### 数据源协议

定义 `BrowserDataProvider` 协议，向调用方提供：

- 当前 Profile 定位。
- 书签搜索。
- 历史搜索。
- 浏览器安装和数据可用状态。
- 使用对应浏览器打开 URL。

首版提供 `ChromiumBrowserDataProvider`。Chrome 与 Edge 共用解析逻辑，仅注入不同浏览器定义。启动器只依赖协议和数据源工厂，不依赖 Chromium 文件格式。

### 当前 Profile

从浏览器用户数据目录的 `Local State` 读取 `profile.last_used`。字段缺失、值为空或目录不存在时回退到 `Default`。本期不合并多个 Profile，也不提供 Profile 设置。

### 书签读取

读取当前 Profile 下的 `Bookmarks` JSON，使用 `JSONSerialization` 或 Codable 结构递归遍历书签树，只收集包含有效 HTTP/HTTPS URL 的 URL 节点。

匹配标题与 URL，忽略大小写。空查询返回有限数量候选；有查询时优先标题前缀和标题包含，其次 URL 包含。书签文件不复制、不修改，也不导入 xxMac 配置目录。

### 历史读取

Chromium 运行时可能锁定 `History` SQLite。每次历史查询先把数据库复制到系统临时目录，以只读方式查询副本，查询完成后删除副本。查询 `urls` 表的标题、URL、访问次数和最后访问时间；按查询词过滤、URL 去重并按最近访问时间排序。

所有磁盘读取和 SQLite 查询在后台队列执行。每次启动器查询带请求标识，较旧查询晚返回时不得覆盖较新的结果。

## 快捷入口与冲突校验

增加统一的功能入口标识，覆盖现有窗口管理、启动器、剪贴板、Snippets、应用启动、快捷指令和浏览器搜索功能。

统一注册中心维护两种互不混用的触发命名空间：

- 键盘组合键：按 Carbon key code 与标准化 modifier flags 查重。
- 启动器关键词：去除首尾空白并进行不区分大小写的查重。

浏览器书签和历史关键词必须满足：

- 非空，且不包含空白字符。
- 两者标准化后不能相同。
- 不能与任何启用的快捷指令关键词相同。

快捷指令新增或修改关键词时，也必须反向检查浏览器关键词。冲突时不保存新值，并在对应输入框下方显示冲突功能名称。加载已有冲突配置时不丢弃数据：保留原值但禁用冲突入口，并在设置页提示用户处理。

统一注册应复用现有持久化结构，避免无必要的数据迁移。现有用户快捷键和快捷指令配置必须能继续解码。

## 设置与持久化

浏览器类型、启用状态、书签关键词和历史关键词保存在配置目录的 `preferences.json`。首次默认值集中放在 `AppDefaultSettings`，与项目现有默认配置约定一致。

浏览器源数据库、书签文件和临时历史副本不属于配置目录内容，不参与配置导出和配置目录迁移。

## 错误处理与隐私

- 只读取用户本机浏览器数据，不上传网络。
- 不修改浏览器文件。
- 历史数据库副本只存在系统临时目录，并保证成功、失败和取消路径均清理。
- URL 仅允许 `http` 和 `https`，忽略浏览器内部页面和无效 URL。
- 文件不存在、JSON 损坏、SQLite 查询失败时返回可诊断状态，不导致启动器崩溃。
- 日志记录浏览器类型和错误类别，不记录搜索词、页面标题或完整 URL。

## 测试与验收

自动化测试至少覆盖：

- 系统默认浏览器识别、安装回退和用户配置覆盖。
- `Local State` 当前 Profile 解析及 `Default` 回退。
- Chrome/Edge 示例书签树递归解析、匹配、排序和无效 URL 过滤。
- 历史数据库副本查询、排序、去重、空查询和临时文件清理。
- 浏览器切换后结果来源和打开目标同步变化。
- 默认及自定义 `bm`、`bh` 激活规则。
- 浏览器关键词彼此冲突、与快捷指令冲突，以及大小写和空白归一化。
- 现有快捷键、应用启动快捷键、剪贴板和 Snippets 快捷键的统一查重。
- 已有偏好配置的兼容加载。

手工验收包括：在 Chrome 和 Edge 分别作为系统默认浏览器的环境首次启动；浏览器运行和退出两种状态下搜索历史；修改关键词后旧关键词立即失效、新关键词立即生效；浅色和深色外观下设置页状态可读。

## 文档更新

同步更新 `README.md`、`README_zh-CN.md`，说明浏览器支持范围、默认与自定义关键词、当前 Profile 规则及本地只读隐私边界。

根据仓库规则同步检查“通用 > 配置目录”说明，明确浏览器书签和历史不会复制到配置目录，也不随配置导出或迁移。

## 非目标

- Safari、Firefox 或其他浏览器的实际数据读取。
- 多 Profile 合并或 Profile 选择 UI。
- 浏览器数据同步、缓存或导入 xxMac 数据库。
- 全文网页内容搜索。
