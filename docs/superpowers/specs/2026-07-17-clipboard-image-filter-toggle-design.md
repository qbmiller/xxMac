# 剪贴板图片筛选开关设计

## 目标

在剪贴板历史 Launcher 顶部搜索栏最右侧增加图片筛选开关，减少用户手动输入图片关键词的操作。

## 交互

- 开关仅在剪贴板历史模式显示，普通 Launcher 和 Snippets 模式不显示。
- 每次打开剪贴板历史时，搜索查询为空，图片开关默认关闭。
- 点击开启时，将搜索框内容固定设置为 `img`，并立即显示全部图片历史。
- 点击关闭时，清空搜索框内容，并恢复显示全部剪贴板历史。
- 用户手动输入 `image` 或 `img` 时，开关自动显示为开启。
- 用户将搜索框修改为除 `image`、`img` 以外的内容时，开关自动显示为关闭。
- 关键词判断忽略首尾空白和大小写，与现有剪贴板搜索规范保持一致。

## 界面

- 使用 SF Symbol 图片图标的按钮式 Toggle，放在顶部搜索栏最右侧。
- 控件使用固定尺寸，避免开关状态变化导致搜索框或 Launcher 布局跳动。
- 开启状态使用现有界面可辨识的选中样式；关闭状态保持低强调。
- 提供简体中文、繁体中文和英文的 tooltip 与无障碍名称。

## 状态与数据流

- 不新增独立持久化设置；这是临时搜索状态，不写入 `preferences.json`。
- 开关状态由当前模式和查询文本派生，避免开关状态与搜索内容不一致。
- Toggle 开启动作通过 `LauncherViewModel` 更新查询为 `img`，复用现有 `$query` 搜索流。
- Toggle 关闭动作通过 `LauncherViewModel` 清空查询，复用现有全量历史查询。
- 数据库与 `ClipboardStorageManager` 的图片查询逻辑保持不变。

## 测试

- 验证剪贴板模式下 `image` 和 `img` 会激活图片筛选状态。
- 验证大小写及首尾空白不影响状态判断。
- 验证其他查询不会激活图片筛选状态。
- 验证开启动作写入固定查询 `img`。
- 验证关闭动作清空查询。
- 运行完整 Swift 测试套件，确认 Launcher、剪贴板和其他工具无回归。

## 非目标

- 不增加图片筛选偏好设置。
- 不改变现有 `photo`、`pictures`、`图片` 等数据库搜索别名的行为；它们仍可按现有逻辑搜索图片，但不会点亮此开关。
- 不增加新的键盘快捷键。
